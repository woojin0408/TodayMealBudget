# Design

## 기준 레퍼런스

- `오늘의밥값.html`: 오늘의 밥값 웹앱 인터랙션/화면 디자인 기준
- `ios-frame.jsx`: iOS 스타일 프레임, 상태바, glass pill, 큰 타이틀, 둥근 리스트 스타일 참고

## 디자인 컨셉

모바일에서도 쓰기 좋지만, 데스크톱에서는 페이지 전체를 활용하는 반응형 웹앱으로 구현한다.

- 감성적인 식비 가계부
- 귀여운 집중 보상 앱
- 숫자가 크게 보이는 예산 카드
- 집중하면 밥값이 오른다는 즉각적인 보상감
- 웹 대시보드 같은 넓은 콘텐츠 배치와 부드러운 카드 UI

## 레이아웃

- 전체 배경은 따뜻한 크림색
- 모바일은 하단 탭을 사용하는 단일 컬럼 구조
- 데스크톱은 좌측 사이드 내비게이션 + 넓은 메인 콘텐츠 구조
- 메인 콘텐츠는 최대 1280px 수준까지 확장
- 데스크톱에서는 카드와 결과 리스트를 2~3컬럼으로 배치
- iPhone 프레임, dynamic island, status bar, home indicator 같은 장식은 사용하지 않는다
- 모바일 하단 탭 높이는 약 72px, safe area 여백 고려

## 타이포그래피

- 기본 폰트는 Pretendard, Apple SD Gothic Neo, system-ui 계열
- 금액 숫자는 tabular-nums 사용
- 핵심 금액은 48~56px 수준으로 크게 표시
- 화면 제목은 26~34px, 800~900 weight 사용
- 보조 텍스트는 12~14px, muted 색상 사용

## 공통 컴포넌트 스타일

### 카드

- 배경: 흰색
- radius: 18~28px
- shadow: `0 2px 16px rgba(0,0,0,0.055)`
- 주요 히어로 카드는 오렌지 그라데이션과 강한 오렌지 그림자 사용

### 버튼

- 큰 터치 영역
- radius: 18~20px
- active 시 scale 0.96 수준
- 메인 버튼은 오렌지 배경과 흰색 텍스트
- 완료/성공 버튼은 초록색 사용

### 하단 탭

- 배경: `rgba(255,255,255,0.92)`
- backdrop blur 적용
- 상단 border는 연한 회색
- 활성 탭은 오렌지
- 아이콘 + 10px 라벨 구조
- 데스크톱에서는 하단 탭 대신 좌측 사이드 내비게이션으로 전환

## 홈 화면

홈은 프로토타입의 핵심 구조를 따른다.

1. 날짜와 앱 이름
2. 오렌지 그라데이션 히어로 카드
3. 오늘 총 밥값 예산을 가장 크게 표시
4. 기본 예산과 보너스를 pill로 표시
5. 남은 밥값/예산 상태 카드
6. 집중 시작/메뉴 추천 CTA 2개
7. 점심/저녁 예산 카드
8. 오늘의 팁 카드

데스크톱에서는 히어로와 상태 카드를 가로로 배치하고, 식비 현황/집중 요약 카드는 여러 컬럼으로 펼친다.

## 집중 화면

- 활동 선택 버튼 3개
- 선택 활동 색상으로 강조
- 적립 중인 금액을 타이머보다 더 크게 표시
- 경과 시간은 보조 정보로 표시
- 실행 중이면 pulse 상태 배지 표시
- 시작, 일시정지, 완료, 취소 버튼 제공

## 결과 화면

- 어두운 히어로 카드에 오늘 성과 표시
- 총 집중 시간, 획득 금액, 총 밥값 예산
- 오늘 먹을 수 있는 대표 메뉴를 큰 이모지로 표시
- 집중 기록 리스트 제공
- 보너스 수준에 따른 동기부여 문구 표시

## 추천 화면

- 질문형 추천 플로우 감성은 유지하되 v1 요구사항의 3개 질문을 사용
- 현재 예산과 티어 문구를 먼저 보여줌
- 추천 메뉴 카드는 오렌지/흰색 카드 조합으로 표시
- 예산을 넘는 메뉴는 “조금 더 집중하면 가능”으로 안내

## 메뉴 리스트

- 금액대 chip과 카테고리 chip을 상단에 배치
- 메뉴는 작은 이모지 썸네일 + 이름 + 가격 + 가능 여부로 표시
- 현재 예산으로 가능한 메뉴는 선명하게, 초과 메뉴는 약하게 표시

## 애니메이션

- 화면 진입: slide-in-up
- 금액 변경: tick
- 추천 결과/대표 메뉴: bounce-in
- 진행 중 상태: pulse-ring
